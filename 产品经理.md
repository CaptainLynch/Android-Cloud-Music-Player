### 验收标准

**F16 歌单代理服务器部署:**
- [ ] 部署 NeteaseCloudMusicApiEnhanced 到公网可访问的服务器（Docker / Node.js）
- [ ] 配置 HTTPS（Nginx 反向代理 + SSL 证书）
- [ ] App 代理模式可成功连接并登录
- [ ] 歌单列表正确展示（包括创建的歌单和收藏的歌单）

- [ ] 「播放全部」功能可用
- [ ] 每日推荐、私人雷达通过代理模式可获取
- [ ] 直连模式仅保留登录验证 + 歌曲 URL 获取，歌单相关接口强制走代理

**BUG-005 搜索平台源过滤:**
- [ ] 选择「酷我」搜索时，结果仅显示 source=kuwo 的歌曲
- [ ] 选择「酷狗」搜索时，结果仅显示 source=kugou 的歌曲
- [ ] 选择「咪咕」搜索时，结果仅显示 source=migu 的歌曲
- [ ] 选择「网易云」搜索时，结果仅显示 source=netease 的歌曲

---

### F15 降级说明

> F15 直连加密经实测，登录 + 每日推荐 + 歌曲 URL 接口可用，但歌单相关接口（user/playlist、playlist/detail、playlist/track/all）不稳定。  
> 决定：歌单功能回退代理模式，直连模式仅保留登录验证 + 每日推荐 + 歌曲 URL。  
> `NeteaseViewModel` 中 `loadUserPlaylists()` 和 `loadPlaylistTracks()` 强制走 proxy，其余方法维持双模式。
