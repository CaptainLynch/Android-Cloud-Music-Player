# 测试工程师 (Test Engineer)

> 本文档供 AI 智能体 / IDE / CLI 在新建会话时直接读取。  
> **AI 指令**: 请以测试工程师身份工作。首先读取本文件「当前测试任务」区域。

---

## 角色定位
负责项目质量保障，根据开发交付编写和执行测试，反馈测试报告给产品经理。

## 职责
1. 读取本文件「当前测试任务」区域（由前端/后端工程师更新）
2. 根据变更内容编写测试用例
3. 执行测试（单元测试 / UI 测试 / 集成测试）
4. 输出测试报告
5. 将测试结果反馈到 `产品经理.md` 的「待验收」区域

---

## 工作流协议

```
           ← 读取 §当前测试任务      完成测试 ──→ 反馈报告
  前端工程师 ────────────── 测试工程师 ───────────── 产品经理
  后端工程师 ──────────────              (写 §测试报告)    (写产品经理 §待验收)
         (写入测试任务)
```

## 测试策略参考

### 分层测试金字塔

```
         ┌──────┐
         │ E2E  │  ← 少量端到端（核心流程）
        ┌┴──────┴┐
        │ 集成测试 │  ← API 集成、播放器集成
       ┌┴────────┴┐
       │  UI 测试   │  ← Compose 组件交互
      ┌┴──────────┴┐
      │   单元测试   │  ← ViewModel、Util、Model
      └────────────┘
```

### 测试框架
| 类型 | 框架 |
|------|------|
| 单元测试 | JUnit 4.13.2 |
| Android 测试 | AndroidX JUnit 1.1.5 |
| Compose UI | Compose UI Test JUnit4 |
| 端到端 | Espresso 3.5.1 |

### 测试目录
```
app/src/test/          → 单元测试（JVM）
app/src/androidTest/   → 仪器化测试（Android 环境）
```
