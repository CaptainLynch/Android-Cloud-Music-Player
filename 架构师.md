## 角色定位
分析需求对系统架构的影响，拆解为前后端开发任务，确保技术方案与现有架构一致。

## 职责
1. 读取 `产品经理.md` 的「当前发布需求」
2. 分析需求对现有架构的影响
3. 将需求拆解为前端任务和后端任务
4. 分别写入 `前端工程师.md` 和 `后端工程师.md` 的「当前任务」区域
5. 若有架构变更需求，在本文件记录设计决策

## 需求分析

> ⚠️ **架构师在此分析需求对架构的影响**

### Part A: 移除直连模式

**背景:**
- F15 网易云原生加密直连功能在迭代 11 中实现
- 实测发现歌单相关接口不稳定，已在迭代 12 中降级为代理模式
- 决定完全移除直连模式，简化架构

**现有架构相关模块:**
- `network/NeteaseApiMode.kt` — PROXY/DIRECT 枚举
- `network/NeteaseCookieJar.kt` — 直连模式 Cookie 管理
- `network/NeteaseDirectInterceptor.kt` — 直连模式拦截器
- `network/NeteaseOriginApiService.kt` — 直连模式 API 接口
- `network/crypto/` — 加密模块 (CryptoUtils, WeApiCrypto, LinuxApiCrypto)
- `data/settings/NeteaseSettings.kt` — directMode 设置
- `ui/netease/NeteaseViewModel.kt` — useDirect 分支逻辑

**分析:**
```
需求 Part A:
- 涉及模块: 网络层（删除加密模块、直连 API、策略模式） + 设置层（移除 directMode） + UI 层（移除切换开关）
- 是否新增模块: 否 — 仅删除
- 是否需要新依赖: 否
- 架构风险: 低 — 纯删除操作，代理模式保持不变
- 与现有架构的一致性: 简化 — 移除策略模式，统一为单一代理模式
- 修改点: 删除 15 个文件，修改 6 个文件
```

### Part B: 心动模式/智能播放

**背景:**
- 新增智能推荐播放功能
- 基于当前歌曲推荐相关歌曲

**现有架构相关模块:**
- `ui/netease/NeteaseViewModel.kt` — 需新增智能推荐逻辑
- `ui/netease/NeteaseScreen.kt` — 需新增智能推荐 UI
- `network/NeteaseApiService.kt` — 需新增智能推荐 API
- `data/model/NeteaseModels.kt` — 需新增数据模型

**分析:**
```
需求 Part B:
- 涉及模块: 网络层（新增 API 接口） + 数据层（新增模型） + UI 层（新增按钮和列表）
- 是否新增模块: 是 — 智能推荐相关代码
- 是否需要新依赖: 否
- 架构风险: 低 — 新增独立功能模块
- 与现有架构的一致性: 一致 — 遵循 MVVM + StateFlow 模式
```

## 架构决策记录

> ⚠️ **如有架构变更，在此记录**

| ID | 日期 | 决策 | 原因 |
|----|------|------|------|
| AD-001 | 2026-05-14 | MVP 阶段不引入 DI 框架 | 项目规模可控，手动注入足够 |
| AD-002 | 2026-05-14 | F10 收藏持久化选用 Room 而非 DataStore | Song 数据结构包含 List<String>，Room + TypeConverter 更适合结构化查询（按ID判断是否已收藏）；DataStore 适合键值对场景 |
| AD-003 | 2026-05-14 | F12 后台播放使用 MediaSessionService 包装现有 MusicPlayerManager | 避免重写全部播放逻辑；MediaSessionService 持有 ExoPlayer，通过 MediaController 桥接 UI 和 Service |
| AD-006 | 2026-05-14 | FavoritesRepository 统一为 Room 单一数据源，移除 SharedPreferences 持久化 | 消除双轨分歧；Room 提供类型安全的结构化查询和 Flow 响应式数据 |
| AD-007 | 2026-05-14 | FavoritesRepository 保持 `object` 单例模式（非 class 构造注入） | 现有多处直接引用，改为 class 需修改 5+ 处代码 |
| AD-008 | 2026-05-15 | F15 歌单降级 — NeteaseViewModel 中 loadUserPlaylists()/loadPlaylistTracks() 强制走代理 | 直连加密在歌单接口上不稳定 |
| AD-009 | 2026-05-15 | BUG-005 客户端侧 source 过滤（非服务端修复） | GD Studio API 服务端过滤不可靠 |
| AD-010 | 2026-05-18 | 移除 F15 直连模式，统一为代理模式 | 直连模式维护成本高，代理模式更稳定可靠 |
| AD-011 | 2026-05-18 | 心动模式/智能播放作为独立功能模块实现 | 与现有播放逻辑解耦，便于后续迭代 |
