**TASK-BE-301: SearchViewModel 搜索结果客户端 source 过滤**

- 涉及文件: `ui/SearchViewModel.kt`
- 问题: GD Studio API 的 `type` 参数传入如 "kuwo" 后，返回结果中仍可能混入 `source="netease"` 的歌曲
- 方案: 在 `searchMusic()` 中将 API 返回值按 `source == selectedPlatform.value` 过滤后再赋值 `_searchResults`
- 修改点:
```kotlin
// 在 searchMusic() 方法中：
val results = RetrofitClient.apiService.searchMusic(...)
val platformValue = platform.value
_searchResults.value = results.filter { it.source == platformValue }
```
- 注意事项: `_searchPlatform.value` 需与过滤条件的 `platformValue` 保持一致
- 预计影响: 低 — 一行 filter，不改变 API 调用、不影响其他逻辑

**验收标准:**
- [ ] 选择「酷我」搜索，结果仅显示 source=kuwo 的歌曲
- [ ] 选择「酷狗」搜索，结果仅显示 source=kugou 的歌曲
- [ ] 选择「咪咕」搜索，结果仅显示 source=migu 的歌曲
- [ ] 选择「网易云」搜索，结果仅显示 source=netease 的歌曲
- [ ] 搜索结果为空时正常显示空状态，无崩溃
