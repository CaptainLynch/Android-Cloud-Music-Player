**TASK-BE-FIX-01: 重写 FavoritesRepository 为 Room 单一数据源**

- 涉及文件: `data/repository/FavoritesRepository.kt`（重写）、`data/local/FavoriteEntity.kt`（新增 toSong/fromSong）
- 问题根因: 当前 FavoritesRepository 使用 SharedPreferences 存储，但 SearchViewModel 中代码按 Room API 风格编写（调用了不存在的构造函数/方法）。同时数据库层（AppDatabase + FavoriteDao）已完整实现但未接入。
- 方案: FavoritesRepository 保持 `object` 单例 + `init(context)` 模式，内部替换为 Room DAO
- 数据流: `init()` → `dao.getAll().collect {}` → 更新 `_favoriteIds` / `_favoriteSongs` MutableStateFlow
- 移除: SharedPreferences 读写代码（KEY_FAVORITE_IDS / KEY_FAVORITE_SONGS / saveToPrefs / loadFromPrefs）
- 新增: FavoriteEntity 追加 `toSong()` 扩展方法和 `companion object { fun fromSong() }` 工厂方法

**验收标准:**
- [ ] FavoritesRepository 内部使用 Room DAO，移除 SharedPreferences
- [ ] `favoriteIds: StateFlow<Set<Long>>` 和 `favoriteSongs: StateFlow<List<Song>>` 行为不变
- [ ] `init(context)` 正确初始化 Room 并启动 Flow 监听
- [ ] `isFavorite(songId)` / `toggleFavorite(song)` API 不变
- [ ] FavoriteEntity.kt 新增 toSong() 和 fromSong() 方法

---

**TASK-BE-FIX-02: 修复 MusicPlayerManager.init() 语法错误**

- 涉及文件: `player/MusicPlayerManager.kt`（第59-70行）
- 问题根因: `fun init()` 内 `.apply {}` 和 `object : Player.Listener {}` 匿名类缺少 3 个闭合 `}`，且丢失了 `onPlaybackStateChanged` 和 `onMediaItemTransition` 监听器
- 方案: 补全缺失的闭合括号，恢复完整的 Player.Listener（含 onIsPlayingChanged、onPlaybackStateChanged、onMediaItemTransition）
- 额外修改: `private var favoritesRepository: FavoritesRepository?` 改为直接引用 `FavoritesRepository` object（不再需要成员变量存储）；移除无效的 `MusicService.start(context)` 调用（此逻辑应在 MediaPlaybackService 中处理）

**验收标准:**
- [ ] `init()` 方法语法正确，Kotlin 编译器无报错
- [ ] Player.Listener 包含完整的 onIsPlayingChanged / onPlaybackStateChanged / onMediaItemTransition
- [ ] 不再持有冗余的 `favoritesRepository: FavoritesRepository?` 引用（直接使用 object）  
- [ ] `toggleFavoriteCurrent()` / `updateFavoriteStatus()` 直接调用 `FavoritesRepository.xxx`

---

**TASK-BE-FIX-03: 修复 SearchViewModel FavoritesRepository 用法**

- 涉及文件: `ui/SearchViewModel.kt`
- 问题根因: 
  - `FavoritesRepository(AppDatabase.getInstance(application))` — object 不能当构造函数调用  
  - `favoritesRepository.allFavorites.map { ... }` — 不存在此属性  
  - `favoritesRepository.isFavoriteFlow(songId)` — 不存在此方法
- 方案:
  1. 移除 `private val favoritesRepository = FavoritesRepository(...)` 构造调用
  2. 在 `init {}` 中调用 `FavoritesRepository.init(application)`
  3. `favoriteSongs` 直接映射 `FavoritesRepository.favoriteSongs`（Room 返回的已是 `List<Song>`，无需 `.map` 转换）
  4. `isFavorite(songId)` 改为调用 `FavoritesRepository.isFavorite(songId): Boolean`
  5. 新增 `val favoriteIds: StateFlow<Set<Long>> = FavoritesRepository.favoriteIds` 暴露给 UI

**验收标准:**
- [ ] SearchViewModel 编译无报错
- [ ] `favoriteIds` 正确暴露为 StateFlow<Set<Long>>
- [ ] `toggleFavorite(song)` 调用 `FavoritesRepository.toggleFavorite(song)`（suspend 函数在 viewModelScope 中 launch）
- [ ] `isFavorite(songId)` 返回 Boolean
- [ ] `favoriteSongs` 映射自 Room DAO Flow
